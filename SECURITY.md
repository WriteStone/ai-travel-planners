## 报告安全漏洞

如果您发现了安全漏洞，请**不要**公开提交 Issue。

请通过以下方式报告：

1. 发送邮件至：your.email@example.com
2. 标题：[SECURITY] 简短描述
3. 包含详细的漏洞信息和复现步骤

我们会在 48 小时内回复并着手处理。

## 安全最佳实践

### API 密钥安全

- ✅ 使用环境变量存储密钥
- ✅ 不要将密钥提交到代码仓库
- ✅ 定期轮换 API 密钥
- ✅ 使用最小权限原则

### 数据安全

- ✅ 启用 HTTPS（生产环境）
- ✅ 使用 Supabase RLS 保护数据
- ✅ 验证所有用户输入
- ✅ 实施速率限制

### 身份验证

- ✅ 使用强密码策略
- ✅ 启用邮箱验证
- ✅ 实施会话超时
- ✅ 防范 CSRF 攻击

## 已知安全考虑

1. **API 密钥暴露风险**: 
   - 客户端密钥存储在 localStorage
   - 建议用户定期更换密钥

2. **第三方服务依赖**:
   - OpenAI API
   - Supabase
   - 高德地图 API

3. **数据隐私**:
   - 用户数据存储在 Supabase
   - 遵循 GDPR 和数据保护法规
